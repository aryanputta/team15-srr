1.5.4.1   CDH Subsystem Overview

The command and data handling (CDH) subsystem provides the command authority, onboard autonomy, telemetry management, science-data storage, and communications interface for the mission. It receives and validates commands from Earth, executes onboard sequences, time-tags the dust, temperature, luminescence, and radiation measurements that make up the primary science return, manages onboard storage, generates telemetry, and routes data to the communications path for return to Earth or to a lunar relay orbiter (NASA 2016). The subsystem hosts the flight software that maintains spacecraft state and preserves command execution through cruise, descent, surface checkout, and permanently shadowed region (PSR) operations, while supporting low-power safe mode and autonomous fault response when real-time command is unavailable (NASA 2009).

The landing site lies at 83.21 degrees south and 203.775 degrees east, on a 4.75 degree slope a few hundred meters from a PSR with a 13.55 degree crater wall (Christensen et al. 2026, via JMARS). Because the lander retains a direct line of sight to Earth at the landing site while the rover loses that line of sight inside the PSR, the governing design drivers are radiation-driven upset risk in the avionics, intermittent communications, and the need to protect buffered data until a contact window opens. Subassembly Descriptions

Flight Computer and Onboard Processing. Executes the flight software and acts as the central authority for state management, command sequencing, fault detection, and data scheduling. Key features are a radiation-tolerant processor protected by a watchdog timer and error-detecting and error-correcting (EDAC) memory. Major components are the single-board computer and processor, volatile working memory with EDAC, and non-volatile boot memory. It interfaces with the power subsystem for regulated power, with every subsystem through the internal data bus, and with the communications interface for command and telemetry exchange.

Data Storage. Buffers science and engineering data between acquisition and the next downlink so no primary data is lost during PSR loss-of-contact. Key features are non-volatile mass memory sized to one full data-generation cycle and explicit priority classes that preserve the highest-value products under storage pressure. Major components are the non-volatile memory module and its EDAC controller. It interfaces with the flight computer, the payload, and the communications interface.

Communications Interface. Accepts validated commands from the receiver and delivers telemetry and buffered science data to the transmitter, framing all traffic in the CCSDS Space Packet Protocol and supporting Proximity-1 for relay operation. Major components are the command and telemetry I/O card and protocol stack. It interfaces with the flight computer and the radio, and is detailed in Section 1.5.4.4.

Internal Data Bus. Carries commands and data between the flight computer and every subsystem and instrument, with redundant routing on the critical command and telemetry paths. Major components are the bus controller and harness. It interfaces with all flight hardware and is captured formally in Section 1.5.8 Interface Control.

Flight Software Architecture. Implements command handling, telemetry generation, state management, data handling, and fault management in a layered structure: a command-validation layer screens incoming traffic, a state manager preserves critical state across resets, and a fault-management layer drives autonomous safe-mode entry. It interfaces with all flight hardware through device drivers and with mission operations through the command and telemetry dictionaries.

1.5.4.2   CDH Subsystem Requirements

Each requirement is a measurable shall statement traced to a parent mission requirement from Section 1.4 and assigned a verification method. 1.5.4.4   Communications Subsystem

The communications subsystem provides the command uplink, telemetry downlink, and science-data return. Because the rover operates inside a PSR with poor or no direct Earth line of sight, the link routes through a lunar relay orbiter across short scheduled windows over the 29.5-day mission. The design must close the relay link with margin under worst-case geometry, use CCSDS and Proximity-1 protocols, and survive 70 K to 395 K, radiation, dust, and landing shock (CCSDS 2020; CCSDS 2013). The rover carries a body-fixed medium-gain antenna to avoid a dust- and cold-vulnerable pointing mechanism, while the lander retains a steerable element for its direct-to-Earth path.

1.5.7.3   CDH Subsystem Redundancy

The flight computer is a block-redundant pair with a cold spare energized after a watchdog-triggered failover, because it is a single point of failure and the 29.5-day intermittent-contact mission allows no ground-commanded swap (NASA 2009). Storage applies error detection and correction and duplicates the highest-priority products until downlink is confirmed. The communications path is functionally redundant through the lander direct-to-Earth path and the relay path, and the internal data bus carries redundant routing on critical command and telemetry lines. On a confirmed critical fault the flight software enters safe mode, preserves state across the reset, records evidence, and fails over to the cold spare, returning the fault log at the next contact window.

Resolved open items   CDH-01 clock accuracy set to +/- 1 ms (oscillator <= 2 ppm, corrected each contact). CDH-02 storage set to >= 4 GB usable / 8 GB provisioned, retaining >= 336 h at the ~80 MB/day payload budget. Verification draws on JMARS contact and shielding geometry, the Phipps et al. 2024 dose maps, and the science-team measurement set defined in the MCR STM.
1.5.8   Interface Control (CDH Interfaces)

The CDH subsystem is the data hub of the spacecraft and defines interfaces with every other subsystem, as shown in Figure 5. It draws regulated power from the power subsystem, issues commands and setpoints to the mechanical mobility and thermal subsystems, collects time-tagged science data from the payload and health telemetry from each subsystem over the internal data bus, and exchanges command and telemetry frames with the communications subsystem for relay to Earth. The four primary flows are power inbound to the avionics, commands outbound from the flight computer, telemetry inbound to the flight computer, and science data inbound to storage. Each interface is governed by the command and telemetry dictionaries and is carried on the redundant critical command and telemetry paths described in Section 1.5.7.3.
